# Security Policy

## Supported Versions

The current supported version line is `0.1.x`.

## Reporting A Vulnerability

Use the repository's private vulnerability reporting channel if you have repository access. If you do not have repository access, contact the package maintainer through the PyPI project contact channel without exploit details, private data, or sensitive logs.

Include:

- Affected LaborSieve version or commit.
- Command used, source type, and report format involved.
- Minimal reproduction steps.
- Expected impact.

Do not include credentials, private job data, or files that contain personal information.

## Review Record

Repository security review findings and remediation notes are recorded in `SECURITY_REVIEW.md`.
